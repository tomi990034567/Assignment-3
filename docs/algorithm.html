The Algorithm

Curating Data and Designing Algorithms
Visit GitHub Page

In the digital age, university websites play a crucial role in providing students with easy access to academic resources. One of the fundamental challenges faced by students is sorting through a vast array of literature. To address this issue, we have developed the Algorithm. The algorithm allows students to search through a database of academic literature relevant to various courses and subjects at the university. When students access the literature search page, they are presented with a number of filter options to refine their results. Only the records that meet all the specified filter conditions are returned as results. This allows students to efficiently narrow down the literature database to find the most relevant resources for their work.

The Features

The algorithm aims to make this course literature list more accessible, by being able to easily search:

  • Author
  • Keyword
  • Book
  • Publishing year

Challenges

The main challenges that we have encountered were related to the algorithm, as we started with relational databases that proved to be hard to link to the algorithm, we tried different examples of code we found in W3schools etc. In the end, we decided to go with a more practical algorithm that just used one dataset in the form of a list where it searches for the relevant words in that one list.

Here are some other challenges with creating the algorithm: - If we only compared the user input with the book titles and author names as they are, the algorithm would be case sensitive, meaning that the same word but with uppercase or lowercase letters would not display all results. To handle this issue we convert both the user input and the book data to lowercase letters before comparing them. - Using xml, csv or json for the data proved difficult so we decided to write the dataset in javascript. - If we compare the user input and the book data as they are, the algorithm will only display results including that exact sequence of characters, meaning that books that include words from the query but in other orders will not be displayed. This also applies to queries that include both the book title and the author names, which wonâ€™t display any results. To handle this issue the user input is split into different words (based on spaces) and stored in an array. The algorithm then checks each word in the array if they are included in the book title or author name. - To ensure that more relevant books are displayed higher up in the result list, the search results are ordered by a score that is based on the number of matching words in the title or the author name.
